SQL Injection

SQL injection é uma vulnerabilidade que se aproveita de falta de sanização ou escape para execução de códigos arbitrários diretamente no banco de dados.

Dependências Maliciosas

Na maioria dos casos, utilizam-se códigos de terceiros como base para o desenvolvimento de aplicações.

Entretanto, em alguns casos, falhas de segurança são inseridas propositalmente nessas biblioteca ou estas são comprometidas, resultando em graves falhas de segurança.

Cópia de códigos

A cópia de trechos de códigos da internet é uma prática comum no mercado de programação. Entretanto, essa ação pode trazer diversos problemas no quesito segurança da aplicação desenvolvida.

Os principais problemas são: licenciamento do código (direitos autorais), introdução de novos bugs e falhas de segurança (deliberadas ou não).

Erros publicamente disponíveis

O tratamento de erro de forma imprópria pode revelar a um atacante informações preciosas para comprometimento da aplicação.

Além, disso há o agravante de que, em alguns casos, essas mensagens podem revelar trechos do código fonte ou outras informações sensíveis como tokens no stacktrace (componentes do erro) para o usuário.
